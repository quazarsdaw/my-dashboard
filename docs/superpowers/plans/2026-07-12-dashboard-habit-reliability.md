# план исправления сохранения заголовка и управления привычками

> **для agentic workers:** required sub-skill: use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. steps use checkbox (`- [ ]`) syntax for tracking.

**цель:** устранить потерю названия дашборда, визуальные артефакты трекера и небезопасные операции с привычками из issue #24, #25, #26 и #27.

**архитектура:** сохранить текущую модель `localstorage` и стабильные идентификаторы привычек. заголовок записывать во время ввода и завершать синхронизацию при фиксации изменения; привычки редактировать внутри существующей липкой ячейки, не пересоздавая объект и его отметки.

**технологии:** статические html/css/javascript, `node:test`, локальная браузерная проверка.

## глобальные ограничения

- не менять определения и условия наград из issue #28
- весь текст в github писать на русском языке в нижнем регистре
- не добавлять зависимости
- не менять идентификатор привычки при переименовании
- не удалять привычку без явного подтверждения
